Linux下搭建hugo博客

0.Hugo简介

​ Hugo是由Go语言实现的静态网站生成器。简单、易用、高效、易扩展、快速部署。

​ 本博客主要参考了B站UP主CodeSheep的视频：https://www.bilibili.com/video/BV1q4411i7gL

1.安装hugo

​ 使用命令sudo apt-get install直接下载安装hugo，或者下载二进制.deb安装程序。 ​ 下载后使用命令sudo dpkg -i hugo_0.80.0_Linux-64bit.deb 完成安装 ​ 安装后查看hugo版本进行验证：

jzhou@jzhou-virtual-machine:~/桌面$ hugo version
Hugo Static Site Generator v0.80.0-792EF0F4 linux/amd64 BuildDate: 2020-12-31T13:37:58Z

2.用hugo生成博客

​ 终端输入命令，在jzhou_blog文件夹中生成一个hugo站点

jzhou@jzhou-virtual-machine:~/桌面$ hugo new site jzhou_blog

​ 得到如下输出后，创建成功

Congratulations! Your new Hugo site is created in /home/jzhou/桌面/jzhou_blog.  

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/ or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.

​ 进入博客根目录，可以看到已经成为了themes、static、layouts、data、content等文件。

3.下载并设置主题

​ 从https://themes.gohugo.io上挑选想要的主题，复制下载命令。进入themes目录下，终端输入

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog/themes$ git clone https://github.com.cnpmjs.org/bjacquemet/personal-web

得到输出

正克隆到 'personal-web'...
remote: Enumerating objects: 345, done.
remote: Total 345 (delta 0), reused 0 (delta 0), pack-reused 345
接收对象中: 100% (345/345), 8.75 MiB | 544.00 KiB/s, 完成.
处理 delta 中: 100% (148/148), 完成.

​ 在进行这一步之前，需要配置好git工具，网上的教程很多，这里就不再介绍了 在使用git clone命令时，经常碰到速度极慢的情况，为此可以使用Github的国内镜像，如github.com.cnpmjs.org，在clone某个项目的时候，可以将github.com替换为github.com.cnpmjs.org。

​ clone后可以看到在themes里多了一个personal-web文件，这说明主题已经下载成功了。

4.在本地启动博客

​ 下载主题后，就可以在本地启动博客了，进入博客的根目录下，执行命令hugo server -t personal-web –buildDrafts ，其中personal-web为主题名，可根据自己的主题名进行更换。

5.写第一篇博客

​ 回到博客的根目录，准备写第一篇博客，生成md文件

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog$ hugo new post/blog.md
/home/jzhou/桌面/jzhou_blog/content/post/blog.md created

生成的blog.md在根目录下的content/post/下 编辑md文件，可以选择一款Marddown编辑器来完成md文件的编辑，这里我使用的是Typora，各位可以根据自己的爱好选择。

6.将博客部署到远端服务器

a.创建仓库

​ 登陆github，点击左上角＋，选择New repository新建一个仓库，再填写仓库的地址，按照昵称全小写.github.io完成创建。（注：必须是昵称的全小写）

b.生成博客页面

​ 使用hugo命令生成静态网页，终端输入（–theme=personal-web 指定主题 –baseUrl=“https://jzhou1997.github.io” 指定仓库地址）

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog$ hugo --theme=personal-web --baseUrl="https://jzhou1997.github.io" --buildDrafts  
Start building sites …

-------------------+-----

  Non-page files   |  0  
  Static files     |  3  
  Processed images |  0

  Sitemaps         |  1

执行完命令后，在根目录下将会生成public文件。

c.将public文件夹创建为git仓库

​ 进入public目录下，初始化git仓库:

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog/public$ git init
已初始化空的 Git 仓库于 /home/jzhou/桌面/jzhou_blog/public/.git/

d.git add *将所有的东西添加进仓库

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog/public$ git add *

e.提交

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog/public$ git commit -m "hugo 第一 次提交"

f.将public文件夹与github上仓库关联

​ 在public目录下，输入命令:

jzhou@jzhou-virtual-machine:~/桌面/jzhou_blog/public$ git remote add origin https://github.com/jzhou1997/jzhou1997.github.io.git
